Property Identification Engine — How It Works

The AI detective process, step by step

Phase 1: Feature Extraction (from listing photos)

Claude Vision analyses each photo and builds a "property fingerprint"

🏠 Exterior Clues
  • House number (if visible on wall/gate)
  • Street name (if visible on sign)
  • Exterior finish — face brick, plaster, painted
  • Roof type — tiles, IBR sheeting, thatch
  • Roof colour — terracotta, charcoal, green
  • Number of storeys
  • Gate/fence style — palisade, wall, precast
  • Garage doors — count, type, colour
🌳 Property Clues
  • Swimming pool — shape (kidney, rectangle, freeform)
  • Large trees — type and position
  • Driveway layout — circular, straight, double
  • Boundary wall height and material
  • Garden features — lapa, braai area, wendy house
  • Solar panels on roof
  • Neighbouring buildings visible
  • Power lines, water towers, landmarks
💡 Quick Wins — Instant Identification

Sometimes the photos give it away immediately:

• Visible house number → almost done, just confirm the suburb
• Street sign in background → massive shortcut
• Unique landmark (church, school, park) → narrows to a block or two
• Estate agent's "Sold" board from previous sale → sometimes visible in old Street View

The AI checks for these first before doing the expensive satellite scan.

Phase 2: Suburb Narrowing

Before scanning satellite images, narrow the search area

Listing Metadata

Property24 says "Queenswood" but agents often list in adjacent "better" suburbs. So we search the listed suburb + all adjacent ones (e.g., Queenswood + Colbyn + Rietondale + Moregloed).

Price + Size Filter

A R2.5M, 3-bed in Queenswood rules out certain streets and plot sizes. Cross-reference with municipal valuation data if available to narrow candidate areas.

Architectural Style

Older face-brick vs modern plaster can indicate which part of the suburb — e.g., houses near the Moot tend to be older 1960s-70s stock vs newer sections.

Phase 3: Satellite Matching

The heavy lifting — scanning aerial imagery for matches

How the scan works:
  1. Divide the target suburb(s) into a grid of tiles (each ~100m x 100m)
  2. Fetch Google Maps Static satellite image for each tile at high zoom (zoom 19-20)
  3. Claude Vision analyses each tile: "Does any property in this tile have a [kidney-shaped pool] + [terracotta roof] + [circular driveway]?"
  4. Matching tiles get flagged as candidates
  5. Zoom in on candidates for detailed comparison
⚠️ Cost Control:
  • Pre-cache suburb tiles — download satellite images once, reuse for all future searches. ~R50 one-time cost per suburb.
  • Smart filtering — if AI says "has a pool", skip all tiles where no pool is visible from satellite. Eliminates 60-70% of tiles immediately.
  • Feature index — over time, build a database: "Tile X has 3 properties with pools, 1 with a thatched roof". Future searches just query the index.

Phase 4: Street View Confirmation

Final verification — does the front of the house match?

For each candidate property from the satellite scan (typically 3-10 matches):

  1. Fetch Google Street View image at the candidate address
  2. Claude Vision compares: "Does this Street View match the front-of-house photo from the listing?"
  3. Score each candidate: boundary wall match, gate style, facade, garage, roof visible from street
  4. Rank by confidence: High (80%+), Medium (50-80%), Low (<50%)

⚠️ Note: Google Street View imagery can be 1-5 years old. Properties may have been renovated, walls repainted, trees grown. The AI accounts for this by weighting structural features (roof, plot shape) higher than cosmetic ones (paint colour, garden).
